## <a name="creating-resource-properties"></a><span data-ttu-id="91460-110">建立資源屬性</span><span class="sxs-lookup"><span data-stu-id="91460-110">Creating resource properties</span></span>

<span data-ttu-id="91460-119">我們使用 `New-xDscResourceProperty` Cmdlet 建立屬性。</span><span class="sxs-lookup"><span data-stu-id="91460-119">To create the properties, we use the `New-xDscResourceProperty` cmdlet.</span></span> <span data-ttu-id="91460-120">下列 PowerShell 命令會建立上述的屬性。</span><span class="sxs-lookup"><span data-stu-id="91460-120">The following PowerShell commands create the properties described above.</span></span>

```powershell

$Ensure = New-xDscResourceProperty –Name Ensure -Type String -Attribute Write –ValidateSet "Present", "Absent"
$DomainCredential = New-xDscResourceProperty –Name DomainCredential -Type PSCredential -Attribute Write
$Password = New-xDscResourceProperty –Name Password -Type PSCredential -Attribute Write
```

## <a name="create-the-resource"></a><span data-ttu-id="91460-121">建立資源</span><span class="sxs-lookup"><span data-stu-id="91460-121">Create the resource</span></span>

<span data-ttu-id="91460-122">資源屬性既已建立，即可呼叫 `New-xDscResource` Cmdlet 來建立資源。</span><span class="sxs-lookup"><span data-stu-id="91460-122">Now that the resource properties have been created, we can call the `New-xDscResource` cmdlet to create the resource.</span></span> <span data-ttu-id="91460-123">`New-xDscResource` Cmdlet 會使用屬性清單作為參數。</span><span class="sxs-lookup"><span data-stu-id="91460-123">The `New-xDscResource` cmdlet takes the list of properties as parameters.</span></span> <span data-ttu-id="91460-124">它也會使用應建立模組位置的路徑、新資源的名稱和包含它的模組名稱。</span><span class="sxs-lookup"><span data-stu-id="91460-124">It also takes the path where the module should be created, the name of the new resource, and the name of the module in which it is contained.</span></span> <span data-ttu-id="91460-125">下列 PowerShell 命令會建立資源。</span><span class="sxs-lookup"><span data-stu-id="91460-125">The following PowerShell command creates the resource.</span></span>

<span data-ttu-id="91460-126">`New-xDscResource` Cmdlet 會建立 MOF 結構描述、基本架構資源指令碼、新資源的必要目錄結構和公開新資源的模組資訊清單。</span><span class="sxs-lookup"><span data-stu-id="91460-126">The `New-xDscResource` cmdlet creates the MOF schema, a skeleton resource script, the required directory structure for your new resource, and a manifest for the module that exposes the new resource.</span></span>

<span data-ttu-id="91460-127">MOF 結構描述檔案位在 `C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.schema.mof`，其內容如下所示。</span><span class="sxs-lookup"><span data-stu-id="91460-127">The MOF schema file is at `C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.schema.mof`, and its contents are as follows.</span></span>

```
[ClassVersion("1.0.0.0"), FriendlyName("Demo_ADUser")]
class Demo_ADUser : OMI_BaseResource
{
  [Key] string UserName;
  [Write, ValueMap{"Present","Absent"}, Values{"Present","Absent"}] string Ensure;
  [Write, EmbeddedInstance("MSFT_Credential")] String DomainCredential;
  [Write, EmbeddedInstance("MSFT_Credential")] String Password;
};
```

<span data-ttu-id="91460-128">資源指令碼位在 `C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.psm1`。</span><span class="sxs-lookup"><span data-stu-id="91460-128">The resource script is at `C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.psm1`.</span></span>

## <a name="updating-the-resource"></a><span data-ttu-id="91460-131">更新資源</span><span class="sxs-lookup"><span data-stu-id="91460-131">Updating the resource</span></span>

<span data-ttu-id="91460-132">如果必須新增或修改資源的參數清單，則可呼叫 `Update-xDscResource` Cmdlet。</span><span class="sxs-lookup"><span data-stu-id="91460-132">If you need to add or modify the parameter list of the resource, you can call the `Update-xDscResource` cmdlet.</span></span> <span data-ttu-id="91460-133">此 Cmdlet 會用新的參數清單更新資源。</span><span class="sxs-lookup"><span data-stu-id="91460-133">The cmdlet updates the resource with a new parameter list.</span></span> <span data-ttu-id="91460-134">如果資源指令碼中已加入邏輯，它會保持不變。</span><span class="sxs-lookup"><span data-stu-id="91460-134">If you have already added logic in your resource script, it is left intact.</span></span>

<span data-ttu-id="91460-135">例如，假設您想要在資源中包含使用者最後一次的登入時間。</span><span class="sxs-lookup"><span data-stu-id="91460-135">For example, suppose you want to include the last log in time for the user in our resource.</span></span> <span data-ttu-id="91460-136">您可呼叫 `New-xDscResourceProperty` 來建立新的屬性，然後呼叫 `Update-xDscResource` 將新屬性新增至屬性清單中，不用再重新撰寫一次資源。</span><span class="sxs-lookup"><span data-stu-id="91460-136">Rather than writing the resource again completely, you can call the `New-xDscResourceProperty` to create the new property, and then call `Update-xDscResource` and add your new property to the properties list.</span></span>

```powershell
$lastLogon = New-xDscResourceProperty –Name LastLogon –Type Hashtable –Attribute Write –Description "For mapping users to their last log on time"
Update-xDscResource –Name 'Demo_ADUser' –Property $UserName, $Ensure, $DomainCredential, $Password, $lastLogon -Force
```
